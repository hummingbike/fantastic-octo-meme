# PRD — AI 기본법 표시 의무 검증 SDK

> 제품명(가칭): **KoAI-Verify**
> 버전: 0.1 (2026-06-06)
> 오너: seokwoo han (1인)

---

## 1. 문제 정의

한국 「AI 기본법」 제31조는 AI 생성 콘텐츠에 표시 의무를 부과한다. 계도기간 종료(~2027.1) 이후 미표시·불충족 표시는 과태료 대상이 된다. 그러나:

- **개발자는 "충족"의 기준을 모른다** — 비가시 워터마크만으로는 불충족(사람 인식 안내 병행 의무).
- **기존 오픈 도구는 검증이 없다** — C2PA 삽입 라이브러리는 많지만 "이 표시가 SNS 재인코딩 후 살아남나"를 판정하는 도구가 없다.
- **표준이 형성 중이다** — TTA 투명성 표준화 진행 중 → 레퍼런스 구현을 먼저 내는 쪽이 표준 권위를 가진다.

## 3. 제품 범위 및 단계

### 3.1 코어 제품 (검증기, Phase 1 필수)

**검증기(Verifier)** — AI 생성 이미지가 제31조를 충족하는지 판정한다.

#### 기능 요건

| ID | 요건 | 우선순위 |
|---|---|---|
| V-01 | C2PA 매니페스트 파싱 및 유효성 확인 | P0 |
| V-02 | EXIF/메타데이터 AI 표시 플래그 판독 | P0 |
| V-03 | 가시 라벨 OCR 탐지(한/영) | P0 |
| V-04 | 흔한 오픈 워터마크 패턴 탐지 | P1 |
| V-05 | 한국법 룰 엔진: 탐지 결과 → 제31조 충족/불충족 판정 | P0 |
| V-06 | 강건성 하니스: 변형 배터리 적용 후 탐지 생존율 측정 | P0 |
| V-07 | 판정 리포트 출력: JSON + 사람이 읽는 요약 | P0 |
| V-08 | CLI 인터페이스: `koai-verify <image>` | P0 |
| V-09 | Python SDK | P0 |
| V-10 | JavaScript/Node.js SDK 래퍼 | P1 |
| V-11 | 웹 플레이그라운드(드래그&드롭) | P1 |
| V-12 | GitHub Action: PR마다 생성물 검증 | P1 |

#### 비기능 요건

- 단일 이미지 검증 응답 ≤ 2초 (로컬 CLI, GPU 불필요)
- 오픈소스(MIT 또는 Apache-2.0), 로컬 실행 가능
- Python 3.10+, 주요 리눅스/macOS 지원
- 탐지 불가 포맷(SynthID 등 비공개)은 명시적 `UNKNOWN` 반환 — 과대주장 금지

### 3.2 강건성 벤치마크 (선점 자산 1호)

변형 배터리 × 탐지율 매트릭스를 **공개 리포트**로 발표한다.

| 변형 유형 | 파라미터 범위 |
|---|---|
| JPEG 압축 | quality 95→50→20 |
| WebP 변환 | quality 90→50 |
| 리사이즈 | 100%→75%→50%→25% |
| 크롭 | 중앙 90%/70% |
| SNS 재인코딩 시뮬 | 인스타그램·트위터·카카오 프로필 시뮬 |
| 스크린샷 캡처 | 96dpi/72dpi 시뮬 |

출력: 포맷별 생존율 히트맵 + 권고 최소 사양.

### 3.3 마커(Marker) — Q3 추가

검증기가 "불충족"이라 지적하는 갭을 해소하는 표시 도구.

| ID | 요건 |
|---|---|
| M-01 | 강건 비가시 워터마크 삽입 (모델 무관, 변형 생존 우선) |
| M-02 | "사람 인식 안내" 자동 동반 (법 요건, 가이드라인 §3 준수) |
| M-03 | 가시 라벨 오버레이 옵션 |
| M-04 | Verify → Mark → Re-verify 폐루프 API |

### 3.4 레지스트리(Registry) — Q3 씨앗, Q4 수익화

발급된 마크를 로그·검증 가능하게 기록. 감사 대시보드(CSV 추출, 증빙 티어) 위에 얹음.

### 3.5 호스팅 API — Q2 씨앗, Q3 유료화

셀프서브 API 키, 사용량 기반 과금. 무료 티어(오픈 SDK) ↔ 유료 티어(호스팅·대량·리포트 보관) 경계.

## 4. 제31조 룰셋 요약 (검증 규칙 기반)

> 원문 근거: 「인공지능 발전과 신뢰 기반 조성 등에 관한 기본법」 제31조 + 시행령 제23조 + 과기정통부 「인공지능 투명성 확보 가이드라인」 (2026.01.22, NIA PDF 원문 확정)
> 상세 명세: [docs/rules/rule_engine_spec_v0.md](docs/rules/rule_engine_spec_v0.md)

| 규칙 ID | 조건 | 판정 |
|---|---|---|
| R-01 | C2PA 매니페스트 존재 + 유효 서명 + AI assertion | 비가시_충족_후보 |
| R-02 | EXIF/XMP AI 플래그 존재 | 비가시_충족_후보 |
| R-03a | 비가시만 있고 외부배포 + 다운로드 1회 안내 없음 | **불충족** |
| R-03b | 비가시만 있고 다운로드 1회 안내 확인 | 충족 |
| R-03c | 비가시만 있고 컨텍스트 불명 | 경고 |
| R-04 | 가시 라벨 "AI 생성" 텍스트 확인 (confidence ≥ 0.80) | 가시_충족_후보 |
| R-05 | 어떤 표시도 탐지 불가 | **불충족** |
| R-06 | 탐지됐으나 강건성 생존율 < 70% | 경고(불안정) |
| R-07 | 딥페이크 서비스 + 비가시 전용 (가시 라벨 없음) | **불충족** |
