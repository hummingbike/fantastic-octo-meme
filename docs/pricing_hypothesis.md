# 사용량 기반 가격 가설 (W21)

> 관련 코드: [`koai_verify/server/pricing.py`](../koai_verify/server/pricing.py)
> 관련 문서: [PRD.md §3.5 호스팅 API](../PRD.md), [PLAN.md W20–22](../PLAN.md)

## 배경

W20에서 호스팅 검증 API v0(`POST /v0/verify`, API 키 인증, 사용량 측정)를
가동했다. PRD §3.5는 호스팅 API를 "Q2 씨앗, Q3 유료화"로 명시하며, 무료
티어(오픈 SDK) ↔ 유료 티어(호스팅·대량·리포트 보관) 경계를 가격 설계의
핵심 과제로 둔다. 아직 외부 통합 파트너의 실사용 데이터가 없으므로(W24
실연동 전), 이 문서는 **검증 가능한 가설**을 세우고 그 가설을 코드(타입이
있는 티어 테이블)로 고정해 이후 실측 데이터로 교정하기 쉽게 만드는 것이
목적이다.

## 가설: 3단계 티어

| 티어 | 월 요청 한도 | 이미지 최대 크기 | 리포트 보관 | 월 기본가 | 초과 단가 |
|---|---|---|---|---|---|
| Free | 500건 | 10MB | 7일 | $0 | — (한도 도달 시 차단) |
| Pro | 20,000건 | 50MB | 90일 | $49 | $0.005/건 |
| Enterprise | 무제한 | 50MB | 365일 | $499 | — |

(코드: `koai_verify.server.pricing.TIER_LIMITS`)

### 산정 근거

- **Free 500건/월**: 개인 개발자가 SDK를 평가·통합 테스트하는 데 필요한
  양으로 가정. CLI/오픈 SDK는 완전 무료(API 미경유)이므로, 호스팅 API의
  Free 티어는 "써보기" 용도이지 상시 운영용이 아니다.
- **Pro 20,000건/월·$49**: 일 평균 ~650건. 초기 채택 파트너(콘텐츠
  플랫폼·소규모 생성 도구) 규모를 가정한 추정치. 건당 환산 시 약
  $0.00245/건으로, 초과 단가($0.005/건)보다 낮게 설정해 한도 내 사용을
  유도한다.
- **Enterprise 무제한·$499**: 대량 트래픽(예: 생성 플랫폼 전체 출력
  검증)이 필요한 파트너 대상. 한도가 없으므로 초과 과금이 없다.
- **이미지 크기 한도**: 현재 `/v0/verify`의 업로드 상한(50MB,
  `app.py::_MAX_UPLOAD_BYTES`)을 Pro/Enterprise 기준으로 그대로 사용하고,
  Free는 10MB로 더 낮게 잡아 대용량 배치 악용을 방지한다.
- **리포트 보관 기간**: 감사 대시보드(Q4, W41–42 "증빙·CSV 유료 티어")로
  이어지는 디딤돌. Free는 단기 평가용(7일), Pro는 분기 단위 점검(90일),
  Enterprise는 연간 감사 대응(365일)을 가정한다.

## 무료 ↔ 유료 경계

- **경계 기준**: 요청 수(월간 누적)이 1차 기준, 이미지 크기·보관 기간은
  2차 기준. R-06(강건성 경고) 등 판정 결과 자체는 가격에 영향을 주지
  않는다 — 표시 의무 판정은 모든 티어에서 동일한 정확도로 제공해야
  신뢰 자산이 훼손되지 않는다 (CLAUDE.md "과대주장은 신뢰 자산을 훼손함"
  원칙과 동일선상).
- CLI·오픈소스 SDK(`pip install koai-verify`, `npm install @koai/verify`)
  로컬 실행은 이 가격 가설과 무관하게 항상 무료다. 가격은 **호스팅 API**
  사용에만 적용된다.

## 코드 인터페이스

```python
from koai_verify.server.pricing import classify_tier, estimate_monthly_cost

classify_tier(15_000)        # PricingTier.PRO
estimate_monthly_cost(25_000, tier=PricingTier.PRO)  # 49.0 + 5000*0.005
```

`GET /v0/pricing` (API 키 인증)은 `get_tier_table()` + 현재 누적 사용량
기준 추천 티어(`usage_to_pricing_recommendation()`)를 반환한다.

## 한계와 검증 계획

- 위 숫자는 **외부 실사용 데이터 없이 세운 가설**이다. 경쟁 벤치마크나
  실측 원가 분석을 거치지 않았으므로 사실로 단정하지 않는다.
- `UsageTracker`는 키별이 아닌 전체 누적치만 집계한다(W20 범위). 키별
  사용량 분리와 정산 주기(매월 리셋) 로직은 아직 구현하지 않았다 — 이번
  주차는 가격 *가설*과 그 가설을 표현하는 자료구조까지만 다룬다.
- 검증 계획: W24 외부 통합 파트너 ≥2곳 실연동 시 실제 요청 분포를 수집해
  한도·가격을 재보정한다. W39("호스팅 API 안정화 + 첫 유료 전환 제안")
  전까지 최소 1회 가설을 갱신한다.
