## 1. 분석 개요

### 1.1 분석 대상

| 구분 | 문서 |
|---|---|
| **KoAI-Verify 룰셋** | `docs/rules/rule_engine_spec_v0.md` (R-01~R-07, v0.2) |
| **한국 AI 기본법** | 「인공지능 발전과 신뢰 기반 조성 등에 관한 기본법」 제31조 (법률 제20676호, 2026.1.22 시행) |
| **시행령** | 시행령 제23조 (고지 및 표시 방법) |
| **가이드라인** | 과기정통부 「AI 투명성 확보 가이드라인」 (2026.01.22, 33p) |
| **TTA 초안 참조** | TTA TC010 SG10-6 작업반 — AI 생성 콘텐츠 투명성 표시 지침 초안 (공개된 논의 방향 기반) |
| **국제 표준** | ISO/IEC 42001:2023, C2PA 2.1, IPTC Photo Metadata 2023.1 |

### 1.2 분석 목적

KoAI-Verify가 실증적으로 확인한 기술적 사실(강건성 벤치마크)을 TTA 표준에 반영하도록 의견을 제출한다.
핵심 기여 포인트: **표시의 생존율(Robustness) 요건**이 현행 초안에 명시되지 않았으며, 실측 데이터로 이 공백을 채울 수 있다.

## 2. 현행 TTA 초안 구조 (공개 논의 방향 기반)

TTA TC010 SG10-6은 ISO/IEC JTC1 SC42와 과기정통부 가이드라인을 기반으로 다음 구조를 논의 중이다.

| 조항 번호 | 제목 | 주요 내용 |
|---|---|---|
| §3 | 용어 정의 | AI 생성 콘텐츠, 딥페이크, 기계 판독 방식, 가시 라벨 |
| §4 | 표시 유형 분류 | 가시(사람 인식) / 비가시(기계 판독) / 혼합 방식 |
| §5 | 표시 방법 요건 | 가시 라벨 최소 요건, 메타데이터 필드 목록 |
| §6 | 딥페이크 강화 요건 | 딥페이크 정의, 가시 표시 강제 |
| §7 | 적합성 평가 | 자가 선언 vs. 제3자 검증 |
| §8 | **강건성 요건** | **미명시 — 현재 공백** |

## 3. KoAI-Verify R-01~R-07과 TTA 초안 비교

### 3.1 전체 매핑 테이블

| KoAI-Verify 룰 | TTA 초안 조항 | 매핑 상태 | 갭 내용 |
|---|---|---|---|
| **R-01** C2PA 매니페스트 유효 서명 | §5.2 비가시 표시 | ⚠️ 부분 정렬 | 서명 유효성 검증 요건 미명시 |
| **R-02** EXIF/XMP AI 플래그 | §5.2 비가시 표시 | ✅ 정렬 | IPTC trainedAlgorithmicMedia 필드 일치 |
| **R-03** 비가시 전용 + 1회 안내 | §5.3 혼합 방식 | ✅ 정렬 | 시행령 제23조 제2항과 일치 |
| **R-04** 가시 라벨 | §5.1 가시 표시 | ✅ 정렬 | "AI 생성" 패턴 목록 보강 필요 |
| **R-05** 표시 전무 → 불충족 | §7 적합성 평가 | ✅ 정렬 | 기본 불충족 판정 일치 |
| **R-06** 강건성 생존율 임계치 | **§8 (미명시)** | ❌ 갭 | **TTA 초안에 강건성 요건 없음** |
| **R-07** 딥페이크 가시 표시 강제 | §6 딥페이크 | ✅ 정렬 | 비가시 단독 금지 조항 일치 |

### 3.2 R-01 상세 갭: C2PA 서명 유효성 검증 미명시

**현황**: TTA 초안 §5.2는 C2PA 매니페스트의 "존재"만 요건으로 하고, 서명 유효성 검증은 명시하지 않는다.

**문제**: 서명 없는 C2PA 매니페스트(unsigned manifest)는 위변조가 용이하다. 표시 의무 이행의 증명력이 낮다.

**KoAI-Verify 대응**: `c2pa_detector.py` — `is_valid_signature == True` AND `has_ai_assertion == True` 이중 검증.

**제안**: §5.2에 "C2PA 매니페스트는 유효한 디지털 서명과 AI 생성 사실 assertion을 포함해야 한다" 조항 추가.

### 3.3 R-06 상세 갭: 강건성 요건 전무 ★ 핵심 기여 포인트

**현황**: TTA 초안 §8 항목이 없거나 "(추후 논의)" 상태이다.

**실측 데이터 (KoAI-Verify 벤치마크 W16)**:

| 변형 유형 | EXIF 생존율 | C2PA 생존율 | OCR(가시 라벨) 생존율 |
|---|---|---|---|
| JPEG 재압축 q=80 | 0% | 0% | 72% |
| Instagram 재인코딩 | 0% | 0% | 68% |
| Twitter/X 재인코딩 | 0% | 0% | 51% |
| KakaoTalk 재인코딩 | 0% | 0% | 44% |
| 리사이즈 50% | 12% | 0% | 91% |
| WebP 변환 q=70 | 0% | 0% | 88% |

> 출처: `benchmarks/results/robustness_matrix.md`

**핵심 발견**:
1. **EXIF/XMP AI 플래그는 SNS 재인코딩 후 100% 소실**된다. 유통 과정에서 비가시 메타데이터는 신뢰할 수 없다.
2. **C2PA 매니페스트 역시 SNS 재인코딩 후 0% 생존**한다. 외부 배포 환경에서 실효성이 없다.
3. **OCR 가시 라벨이 가장 높은 생존율**을 보이지만, KakaoTalk 환경에서 44%로 급락한다.

**제안 조항 (§8 신설)**:

```
§8 표시의 지속성(강건성) 요건

8.1 사업자는 AI 생성 표시가 일반적인 유통 변형 과정(압축·리사이즈·재인코딩)에서도
    이용자에게 인지 가능한 상태로 유지될 수 있도록 표시 방식을 선택해야 한다.

8.2 비가시 표시(C2PA 매니페스트, EXIF/XMP 메타데이터)만을 적용하는 경우,
    해당 표시가 SNS 등 외부 유통 채널에서 소실될 수 있음을 인지하고,
    가시 라벨 또는 복수 표시 방식 적용을 권장한다.

8.3 강건성 검증 도구(예: KoAI-Verify 강건성 하니스)를 활용하여
    표시 생존율 70% 이상 유지를 목표로 한다. (권고 사항)
```

## 4. 국제 표준과의 정렬 상태

| 국제 표준 | KoAI-Verify 관련 룰 | 정렬 상태 |
|---|---|---|
| **C2PA 2.1** §3.2 AI Assertions | R-01 C2PA 검증 | ✅ C2PA 검증 로직이 C2PA 2.1 스펙 기반 |
| **IPTC Photo Metadata** — `DigitalSourceType: trainedAlgorithmicMedia` | R-02 EXIF AI 플래그 | ✅ 동일 필드 사용 |
| **ISO/IEC 42001:2023** §6.1 위험 평가 | R-06 강건성 경고 | ⚠️ 42001은 조직 수준 관리, 표시 강건성은 미정 |
| **ISO/IEC 42005** (진행 중) 영향 평가 | R-07 딥페이크 | ⏳ 표준 완성 후 재검토 필요 |
