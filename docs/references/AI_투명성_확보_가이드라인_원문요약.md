## 1. 법적 근거

**「인공지능 발전과 신뢰 기반 조성 등에 관한 기본법」** (법률 제20676호, 2025.1.21 제정, 2026.1.22 시행)

### 제31조 인공지능 투명성 확보 의무

| 항 | 적용 조건 | 의무 내용 |
|---|---|---|
| **제1항** | 고영향 AI 또는 생성형 AI를 이용한 제품·서비스 제공 시 | AI 기반 운용 사실을 이용자에게 **사전 고지** |
| **제2항** | 생성형 AI 및 이를 이용한 제품·서비스 제공 시 | 결과물이 AI로 생성되었다는 사실을 **표시** |
| **제3항** | AI 시스템으로 실제와 구분하기 어려운 가상의 생성 결과물 제공 시 | 사실을 이용자가 명확하게 알 수 있도록 **고지 또는 표시** |

### 시행령 제23조 — 고지 및 표시 방법

**제1항 — 사전고지 방법 (제31조 제1항 이행)**
1. 제품에 직접 기재하거나, 계약서·사용설명서·이용약관 등에 기재
2. 이용자의 화면 또는 단말기 등에 표시
3. 제품·서비스를 제공하는 장소에 인식하기 쉬운 방법으로 게시
4. 그 밖에 제품·서비스의 특성을 고려하여 과학기술정보통신부장관이 인정하는 방법

**제2항 — 표시 방법 (제31조 제2항 이행)** ★ 핵심
1. 표시 방법은 **사람이 인식할 수 있는 방법**과 **기계가 판독할 수 있는 방법**이 있음
2. **기계가 판독할 수 있는 방법을 사용할 경우에는 최소 1회 이상 안내 문구·음성 등을 제공해야 함**

**제3항 — 딥페이크 표시 시 고려사항 (제31조 제3항 이행)**
1. 이용자가 시각·청각 등을 통하거나 소프트웨어 등을 이용하여 쉽게 내용을 확인할 수 있는 방법
2. 주된 이용자의 연령, 신체적·사회적 조건 등을 고려하여 고지 또는 표시할 것

## 2. 적용 대상

**의무 있음**: 이용자에게 최종적으로 AI 제품·서비스를 제공하는 **인공지능사업자**
- AI개발사업자: AI를 직접 개발하여 제공하는 자 (예: 네이버, OpenAI)
- AI이용사업자: AI를 이용하여 제품·서비스를 제공하는 자 (예: 뤼튼테크놀로지스)
- 해외 사업자라도 우리 국민에게 영향을 미치면 적용 대상

**의무 없음**: 단순히 AI 결과물을 자신의 서비스에 활용하는 자
- 예: AI로 CG를 생성하여 영화에 삽입한 영화제작자

## 3. 서비스 내 vs. 외부 반출 — 표시 기준 차이 ★ 검증기 핵심 판정 기준

| 구분 | 표시 위치 | 방법 |
|---|---|---|
| **서비스 이용 환경 내** (UI, 플랫폼 내) | 결과물 내 OR 서비스 UI | 사람 인식 OR 기계 판독 + 1회 이상 안내 |
| **외부 반출** (다운로드·공유) | **결과물 자체에 반드시 표시** | 사람 인식 OR 기계 판독 + 1회 이상 안내 |

> 서비스 내에서 UI 표시를 했더라도, **다운로드·공유 기능 제공 시에는 반드시 결과물 내에 표시**해야 함.

## 4. 콘텐츠 유형별 외부 반출 표시 방법

### 이미지 생성물 ★ KoAI-Verify 주 대상

| 방법 | 내용 | 비가시 단독 사용 조건 |
|---|---|---|
| **사람이 인식 (가시)** | 이미지 내 로고 삽입 등 가시적 방법 | 단독 가능 |
| **기계 판독 (비가시)** | 디지털 워터마킹, 메타데이터 | **다운로드 시 1회 이상 안내 문구·음성 필수** |
| **딥페이크** | 사람이 직관적으로 인식할 수 있는 방법 **만** | 비가시 단독 불가 |

### 동영상 생성물
- 가시: 화면 영역 일부에 로고, 영상 시작 부분에 AI 생성 사실 안내
- 딥페이크: AI로 생성된 재생구간 전체에 로고 표출 필수
- 비가시: 다운로드 시 1회 이상 안내

### 음성 생성물
- 가시: 음성 시작 부분에 AI 생성 사실 안내 멘트
- 비가시: 음성 워터마킹, 메타데이터 + 다운로드 시 1회 이상 안내

### 텍스트 생성물
- 파일 형태: 문서 머리말, 파일 메타데이터로 표시 + 다운로드 시 1회 이상 안내
- 코드: 주석 또는 .md 파일 명시

## 5. 예외 및 완화 조건

### 딥페이크 예술적·창의적 표현물 예외
- 미술·영화·문학·사진·출판·만화·게임·애니메이션 등 창의적 표현 활동의 결과물
- 전시·향유를 저해하지 않는 방법으로 표시 가능 (완화된 기준 적용)
  - 예: 시간적으로 주요 콘텐츠와 차이를 두어 표시, 비가시적 표시 방법 적용 등

### 중간 생성물 제외
- 프롬프트 입력·생성 이미지 선택 등 편집 과정의 중간 생성물에는 표시 의무 없음
- **결과물 제공(추출·저장) 단계에서만 표시**

### 단순 편집 제외
- 생성형 AI 본질적 기능이 사용되지 않은 경우 (예: 이미지 자르기만 사용)

### 클립보드 복사 제외
- 이용자가 클립보드(Copy & Paste) 기능을 이용하는 경우는 해당하지 않음

## 7. 룰 엔진 반영 핵심 교정 사항

> 이 섹션은 가이드라인 원문 확인 후 rule_engine_spec_v0.md에서 수정된 내용 요약.

### R-03 수정: "비가시 단독 → 불충족"은 **부분적으로만 맞음**

원문 정확한 해석:
- 비가시(기계 판독) 방식 단독 사용 = **1회 이상 안내 문구·음성 필수**
- 안내 문구가 있으면 → **충족**
- 안내 문구가 없으면 → **불충족**

즉, **비가시 + 1회 안내 = 충족** / **비가시만 = 불충족**

검증기 판정 한계:
- 이미지 파일 단독으로는 "다운로드 시 1회 안내" 여부를 알 수 없음
- → 비가시 전용 탐지 시 `WARNING` (컨텍스트 불명) 반환이 적절
- → 서비스 컨텍스트 정보 제공 시 정확한 판정 가능

### 딥페이크 (R-07 신설 필요)
- 딥페이크는 **사람이 인식할 수 있는 방법만** 허용 (비가시 단독 불가)
- 가이드라인 §4 이미지 생성물 딥페이크: "명확하게 인식할 수 있는 방법으로써 사람이 인식할 수 있는 방법만을 적용"
